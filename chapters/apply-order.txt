      hardcoded to THIS cluster's Grafana route (apps.cluster-mxqd5.dynamic2...) — every
      new cluster gets a different *.apps.<id>.<domain> hostname. Update it to match
      `oc get route grafana -n grafana` after Chapter 7 runs (Kiali itself works fine
      with the stale value — it only breaks the optional embedded Grafana link).

      --no-check-certificate \

CHAPTER 06 — CI/CD
========================================

    -- Enable the Pipelines console plugin (cluster-admin, one-time) --
    -- The operator installs the pipelines-console-plugin pod but doesn't
    -- always enable it in the Console config — without this, Developer
    -- perspective's Pipelines view stays empty even though `oc get pipeline`
    -- shows the Pipeline exists --
    oc get console.operator.openshift.io cluster -o jsonpath='{.spec.plugins}'
    -- if "pipelines-console-plugin" is missing from that list: --
    oc patch console.operator.openshift.io cluster --type=json \
      -p '[{"op": "add", "path": "/spec/plugins/-", "value": "pipelines-console-plugin"}]'

    -- Enable the GitOps console plugin too (same gap, same fix) --
    -- Without this, ArgoCD shows nothing in the OpenShift console even
    -- though `oc get application -n openshift-gitops` shows it Synced/Healthy --
    oc patch console.operator.openshift.io cluster --type=json \
      -p '[{"op": "add", "path": "/spec/plugins/-", "value": "gitops-plugin"}]'

    -- Grant financeflow-cicd permission to push images and run buildah --
    -- (financeflow-cicd ServiceAccount already exists from Chapter 4) --
    -- Without these two, the PipelineRun's build Task fails (no registry
    -- push permission) or every Task in the Pipeline gets "Permission
    -- denied" writing to the shared pipeline-source PVC — pipelines-scc,
    -- NOT privileged, since its fsGroup: MustRunAs is what makes the shared
    -- workspace writable across Tasks/pods; see Chapter 6 lab Troubleshooting --
    oc adm policy add-role-to-user \
      registry-editor \
      system:serviceaccount:financeflow-workshop:financeflow-cicd
    oc adm policy add-scc-to-user pipelines-scc \
      -z financeflow-cicd \
      -n financeflow-workshop

    -- Create the GitHub webhook Secret imperatively --
    oc create secret generic github-webhook-secret \
      --from-literal=secret=$(openssl rand -hex 20)

    -- Create the git push credentials Secret and link it to financeflow-cicd --
    -- (needed by task-update-manifest.yaml's git-commit-push step — see
    --  secret-git-credentials.yaml comments for why no Task/workspace change
    --  is needed) --
    oc create secret generic git-credentials-cicd \
      --type=kubernetes.io/basic-auth \
      --from-literal=username=<your-github-username> \
      --from-literal=password=<your-github-PAT-with-repo-scope>
    oc annotate secret git-credentials-cicd tekton.dev/git-0=https://github.com
    oc secrets link financeflow-cicd git-credentials-cicd

    -- Label the namespace for ArgoCD (NEVER `oc apply -f` namespace-argocd-managed.yaml
    -- as a full object — see its comments: that would wipe Chapter 5's
    -- istio-injection label via kubectl's last-applied-configuration 3-way merge) --
    -- Without this label, the GitOps operator never creates the RoleBinding that
    -- lets ArgoCD's application-controller manage Deployments/Services/HPAs here;
    -- sync fails with "is forbidden: ... cannot patch resource ..." --
    oc label namespace financeflow-workshop argocd.argoproj.io/managed-by=openshift-gitops
